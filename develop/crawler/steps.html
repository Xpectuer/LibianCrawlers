Steps API ​

在工程目录下的 ./steps 目录中存放着各个网站的自动化脚本。

API 指南 ​

杂项 ​

异常处理 ​

每个形如 {"fn":"function_name"} 的指令都可以传入 on_success_steps 与 on_timeout_steps 属性，例如:

json
{
  "fn": "page_random_mouse_move",
  "on_success_steps": "continue",
  "on_timeout_steps": "continue"
}

这两个属性的值可以传入 StepsBlock。

on_timeout_steps 将会在函数抛出 常见的超时异常 时发生作用。

常见的超时异常如 playwright.async_api.TimeoutError,asyncio.TimeoutError, TimeoutError 。

  • 当未指定 on_timeout_steps 时，异常将会继续抛出，这很有可能会直接中止爬虫。
  • 当指定 on_timeout_steps 后。将会继续执行 on_timeout_steps 值的指令。

而 on_success_steps 会于没有任何异常抛出时执行，通常用作 on_timeout_steps 的 else 分支

description ​

每个形如 {"fn":"function_name"} 的指令都可以传入 description 属性，例如:

json
{
  "fn": "page_random_mouse_move",
  "description": "正在模拟人类移动鼠标"
}

description 属性不会起任何作用，它仅用于便于人类阅读。

Step 指令列表 ​

continue

continue 指令不会起任何作用，通常作为回调块的占位符使用，以代替回调块的默认处理。

比如，很多时候可以在 on_timeout 回调命令中使用，来阻止默认的抛出超时异常的行为。

json
{
  fn: "page_click",
  args: [
    ".cpCloseIcon",
  ],
  on_timeout_steps: "continue"
}

page_click

封装了以多种方式点击页面的命令。共有以下模式:

  • A: 使用 page.mouse.click 并指定 xy 坐标直接点击页面。

  • B: 先使用 page.locator 定位元素，然后:

    • B1: 使用 Locator.click 触发点击事件。

    • B2: 使用 Locator.tap 触发点击事件。

TIP

由于这里封装了多种模式的 kwargs 参数，因此在下方的参数含义文档中会特地注明某可选参数适用的模式。

name require type example
arg[0] ✔️ Union[string, XY]
timeout number 6000
method Literal["click", "tap"]
wait_any_page_create_time_limit number 3000
on_new_page Union[Literal["ignore", "switch_it_and_run_steps_no_matter_which_page"], StepsBlock]
has_text string
has_not_text string
on_locator OnLocatorBlock
modifiers List[Literal["Alt", "Control", "ControlOrMeta", "Meta", "Shift"]]
force boolean
no_wait_after boolean
trial boolean
position XY
delay number
button Literal["left", "middle", "right"]
click_count number
参数含义文档

arg[0]

传入字符串时，将会使用【模式B】，并将其作为传递给 _page.locator 的 selector 参数。

传入 XY 时，将会使用【模式A】，并将其作为传递给 page.mouse.click 的参数。

timeout

【适用模式B】。超时时间。 默认值 5000，单位为毫秒。

method

【适用模式B】。

传入 默认值 "click" 时，将会使用【模式B1】，使用 playwright 的 Locator.click 触发点击事件。

传入 "tap" 时，将会使用【模式B2】，使用 playwright 的 Locator.tap 触发点击事件。

wait_any_page_create_time_limit

【适用全部模式】。当在点击页面后的这段时间内有新页面被创建，且 on_new_page 属性不为 "ignore"，则会调用 on_new_page 块；反之则不会调用。 默认值 3000。单位为毫秒。

on_new_page

【适用全部模式】。此参数用于控制 “当点击页面后假如出现新页面” 的行为：

默认值 "switch_it_and_run_steps_no_matter_which_page" 将会自动将当前页面指针指向新页面，但假如没有新页面则无事发生。

"ignore" 值将会忽略新页面（且当前页面指针不会指向新页面）。

如果传入 StepsBlock，则先将自动将当前页面指针指向新页面，然后 在新页面中调用此 StepsBlock ，并且会在 StepsBlock 块完毕后自动将当前页面指针 指回旧页面 。

has_text

【适用模式B】。传递给 Page.locator 的 has_text 可选参数。根据包含的文本查找元素，可以通过子串、精确字符串或正则表达式进行匹配。匹配元素内部包含指定文本的元素，可能是子元素或后代元素。当传递字符串时，匹配不区分大小写并搜索子字符串。例如，"Playwright" 匹配 <article><div>Playwright</div></article> 。

has_not_text

【适用模式B】。传递给 Page.locator 的 has_not_text 可选参数。匹配内部某处不包含指定文本的元素，可能是子元素或后代元素。传递字符串时，匹配不区分大小写并搜索子字符串。

on_locator

【适用模式B】。在每次返回 Locator 后继续调用

modifiers

【适用模式B】。传递给 Locator.click 或 Locator.tap 的 modifiers 可选参数。

force

【适用模式B】。不进行 actionability 检查而强行点击。

no_wait_after

【适用模式B】。此参数已经被 playwright 弃用。

trial

【适用模式B】。设置后，此方法仅执行 actionability 检查并跳过操作。默认为 false。等待元素准备好执行操作而不执行操作很有用。请注意，无论 trial 如何，键盘 修饰键 都会被按下，以允许测试仅在按下这些键时才可见的元素。

position

【适用模式B】。传递给 Locator.click 或 Locator.tap 的 position 可选参数。

delay

【适用模式A和B】。

在【模式A】中，作为传递给 page.mouse.click 的 delay 可选参数， 默认值 0。

在【模式B】中，作为传递给 Locator.click 或 Locator.tap 的 delay 可选参数， 默认值 0。

button

【适用模式A和B1】。

在【模式A】中，作为传递给 page.mouse.click 的 button 可选参数。 默认值 "left"。

在【模式B1】中，作为传递给 Locator.click 的 button 可选参数。 默认值 "left"。

click_count

【适用模式A和B1】。

在【模式A】中，传递给 page.mouse.click 的 click_count 可选参数。 默认值 1。

在【模式B1】中，传递给 Locator.click 的 click_count 可选参数。 默认值 1。

示例

json
{
  "fn": "page_click",
  "args": [
    ".closeIcon"
  ],
  "kwargs": {
    "timeout": 2000
  },
  "description": "如果有广告弹窗则关闭",
  "on_timeout_steps": "continue"
}

page_scroll_down

将页面向下滚动到底或最大高度。

当滚到底部时，会上滚一下再下滚，反复多次后若高度不变（没有加载新玩意）则认为滚动完成。

如果指定了 page_click_if_found 属性，则会在合适的时机点击每个符合选择器的元素，在点击之后还会运行 dump_page ，并可以指定 dump_page 前后运行指定的步骤。

name require type example
max_height number 20000
delta_y number 233
interval number 0.5
retry_scroll_down_limit number 2
retry_scroll_up_limit number 2
page_click_if_found object
参数含义文档

max_height

向下滚动到最大高度后将会停止。 默认值 20000 。如果你需要完整的网页截屏，请不要设置过高的值，因为 firefox 有个 老问题 ，设置过高的高度可能导致截屏失败。

delta_y

每次向下滚动的距离因子。实际向下滚动的距离为其的 0.3~1.6 倍。 默认值 233.0 。建议不要修改，容易出 BUG。

interval

每次向下滚动事件之间的时间间隔因子。实际的间隔时间为它的 0.7~1.3 倍。 默认值 0.5。建议不要修改，容易出 BUG。

retry_scroll_down_limit

默认值 2 。当发现滚动后高度和上次一致时，再向下滚动此次数，若还是一致或高度缩短，则认为 没有发现新加载的内容。

retry_scroll_up_limit

默认值 2 。当 没有发现新加载的内容 时，会 试图向上滚动再向下滚动检查 此次数。当该过程中发现了新加载的内容，清空此计数器并继续向下滚动；直到此计数器超过该值 或 超过高度上限 时，才会认为页面已经滚动到底。

page_click_if_found

schema
json
{
  "type": "object",
  "description": "当指定此值时，将会在向下滚动的过程中不断的将 满足 `locator` 属性的选择器要求的元素 的位置收集入列表中。\n\n此列表会根据元素的 inner_text 与 inner_html 去重。\n\n仅当同时满足 元素可以被点击、元素出现在视口内、元素的长宽均超过5 时，将会使用 [【模式A】] （即传入XY）点击(#page_click) 此元素。如果没满足点击条件，则会保留在列表中，直到满足条件才被点击并移出列表。\n\n对于每个被点击的元素，点击元素并短暂的等待后: \n\n1. 先会执行一次 dump_page ;\n\n2. 然后执行 `on_found_after_click_steps` 属性中的 StepsBlock（通常可以用作继续向下滚动）;\n\n3. 再运行一次 dump_page;\n\n4. 最后再执行 `on_found_after_click_steps` 属性中的 StepsBlock（通常可以用作回到主页）。 ",
  "minProperties": 1,
  "required": [
    "locator"
  ],
  "properties": {
    "locator": {
      "type": "string",
      "description": "选择器。"
    },
    "on_found_after_click_steps": {
      "$ref": "#/definitions/StepsBlock"
    },
    "on_found_after_click_and_dump_steps": {
      "$ref": "#/definitions/StepsBlock"
    }
  }
}

当指定此值时，将会在向下滚动的过程中不断的将 满足 locator 属性的选择器要求的元素 的位置收集入列表中。

此列表会根据元素的 inner_text 与 inner_html 去重。

仅当同时满足 元素可以被点击、元素出现在视口内、元素的长宽均超过5 时，将会使用 [【模式A】] （即传入XY）点击(#page_click) 此元素。如果没满足点击条件，则会保留在列表中，直到满足条件才被点击并移出列表。

对于每个被点击的元素，点击元素并短暂的等待后:

  1. 先会执行一次 dump_page ;

  2. 然后执行 on_found_after_click_steps 属性中的 StepsBlock（通常可以用作继续向下滚动）;

  3. 再运行一次 dump_page;

  4. 最后再执行 on_found_after_click_steps 属性中的 StepsBlock（通常可以用作回到主页）。

示例

json
{
  "fn": "page_scroll_down"
}
